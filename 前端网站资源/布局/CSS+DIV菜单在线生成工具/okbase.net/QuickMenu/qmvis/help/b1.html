About The Core Menu
Welcome to QuickMenu! The menus produced by this visual interface are 100% Pure CSS with extra features for browsers with JavaScript enabled.  We often get asked...  I thought your product was pure CSS, yet the saved menu includes JavaScript?

This is a common misunderstanding with QuickMenu, the menus by default do save with JavaScript, however they are designed to function in browsers with JavaScript disabled, or with the QuickMenu saved JavaScript removed from the document.  QuickMenu combines the extended features only afforded by script, such as...  delay timers for user control, IE6 - IE5 support, persistent parent states, cross browser animations, browser bug fixes, style enhancements, and more with a true 100% Pure CSS menu for JavaScript disabled browsers.

With this menu concept the developer can choose between 100% Pure CSS for all users by simply removing the JavaScript, or a combination menu that takes full advantage of a given browser's script capabilities.  Both menu types pool from the same CSS styles and HTML structure, allowing for cross-code performance with virtually no overhead. 

CSS styles are defined and function identically in both script-less CSS and JavaScript enhanced menus.  Both menu code types also share identical orientation options, dividers, titles, individual-inline styles, hover styles, auto sub positioning, sub position offsets, plus more.

The core QuickMenu combines the Internet's most extensive feature set into the world's most compact and fastest loading, and running menu. Weather designed by hand or though the extensive visual interface, each custom QuickMenu is pure though and though, as if hand coded for your specific web site.
