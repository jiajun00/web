Validation
QuickMenu uses 100% fully valid XHTML regardless of add-ons employed or styles applied.

The only portion of known non XHTML/Strict code produced by QuickMenu is the no script access option when saving a hybrid style menu (by default this option is not enabled). The no script tag is not valid under this doctype, while the menu remains completely functional, the standards do not account for advanced scenarios where this type of use should be valid.

If you absolutely require strict validation, do not include the no script access option when publishing a hybrid menu.

Please do not test validation for the menu by checking the OpenCube web site, while we use the valid QuickMenu software on our site, we cannot guarantee that the rest of our HTML code adheres to the same strict guidelines.
